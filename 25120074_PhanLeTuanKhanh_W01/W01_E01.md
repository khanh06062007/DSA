## Exercise 1: Linked List

### Context

Imagine a small clinic in a busy neighborhood. Every day many patients arrive and wait to see the doctor. The clinic is simple and does not have a complicated computer system. Instead, the staff wants a small program that can **keep track** of patients who are waiting.

When a patient arrives, the staff needs to record the patient in a **waiting list**. Most patients join **the end of the list** and wait for their turn. Sometimes an **emergency patient** arrives. In that situation the patient must be placed at **the front of the list** so the doctor can see them first.

As the doctor finishes seeing patients, the first patient in the list leaves the waiting area. The staff also wants the ability to see the full list of patients at any time. In a few special situations a patient may need to be placed after another specific patient.

Your task in this lab is to design a program that can manage this waiting list. The list must grow and shrink as patients arrive and leave. Instead of using an array, we will build the system using a **singly linked list**. This structure allows us to add and remove patients efficiently while the program is running.

#include <iostream>
#include <string>

struct PatientNode {
    int id;
    std::string name;
    PatientNode* next;
};

PatientNode* createNode(int id, std::string name) {
    PatientNode* newNode = new PatientNode;
    newNode->id = id;
    newNode->name = name;
    newNode->next = nullptr;
    return newNode;
}

void insertHead(PatientNode*& head, PatientNode* newNode) {
    newNode->next = head;
    head = newNode;
}

void insertTail(PatientNode*& head, PatientNode* newNode) {
    if (head == nullptr) {
        head = newNode;
        return;
    }
    PatientNode* current = head;
    while (current->next != nullptr) {
        current = current->next;
    }
    current->next = newNode;
}

int countPatients(PatientNode* head) {
    int count = 0;
    PatientNode* current = head;
    while (current != nullptr) {
        count++;
        current = current->next;
    }
    return count;
}

PatientNode* findMiddle(PatientNode* head) {
    if (head == nullptr) {
        return nullptr;
    }

    PatientNode* slow = head;
    PatientNode* fast = head;

    while (fast != nullptr && fast->next != nullptr) {
        slow = slow->next;
        fast = fast->next->next;
    }

    return slow;
}

void removeByID(PatientNode*& head, int id) {
    PatientNode* current = head;
    PatientNode* previous = nullptr;
    while (current != nullptr) {
        if (current->id == id) {
            if (previous == nullptr) {
                head = current->next;
            } else {
                previous->next = current->next;
            }
            delete current;
            return;
        }
        previous = current;
        current = current->next;
    }
}

void removeHead(PatientNode*& head) {
    if (head == nullptr) {
        return;
    }
    PatientNode* temp = head;
    head = head->next;
    delete temp;
}

void reverseList(PatientNode*& head) {
    PatientNode* prev = nullptr;
    PatientNode* current = head;
    PatientNode* next = nullptr;
    while (current != nullptr) {
        next = current->next;
        current->next = prev;
        prev = current;
        current = next;
    }
    head = prev;
}

void printList(PatientNode* head) {
    PatientNode* current = head;
    while (current != nullptr) {
        std::cout << "ID: " << current->id << ", Name: " << current->name << std::endl;
        current = current->next;
    }
}

int main() {
    PatientNode* head = nullptr;

    insertTail(head, createNode(101, "Alice"));
    insertTail(head, createNode(102, "Bob"));
    insertTail(head, createNode(103, "Charlie"));
    
    std::cout << "After adding 3 patients:" << std::endl;
    printList(head);
    std::cout << "Total patients: " << countPatients(head) << std::endl << std::endl;

    insertHead(head, createNode(104, "Emergency David"));
    std::cout << "After emergency patient arrives:" << std::endl;
    printList(head);
    std::cout << std::endl;

    removeHead(head);
    std::cout << "After first patient seen:" << std::endl;
    printList(head);
    std::cout << std::endl;

    PatientNode* middle = findMiddle(head);
    if (middle != nullptr) {
        std::cout << "Middle patient: " << middle->name << std::endl << std::endl;
    }

    removeByID(head, 102);
    std::cout << "After removing patient ID 102:" << std::endl;
    printList(head);
    std::cout << std::endl;

    reverseList(head);
    std::cout << "After reversing the list:" << std::endl;
    printList(head);

    while (head != nullptr) {
        removeHead(head);
    }

    return 0;
}

## Extension: Beyond Singly Linked Lists

While the singly linked list you built for the clinic is a powerful foundational data structure, many real-world scenarios require more complex operations. Below is a brief introduction to advanced data structures that build directly upon the linked list concept.

### 1. Doubly Linked List
* **Structural Difference:** Each node contains 3 parts: the data, a pointer to the `next` node, and a new pointer to the `previous` node.
* **Use Case:** Applications requiring two-way navigation. 
  * A music playlist (next/previous song)
  * Web browser history (back/forward buttons)
  * Undo/redo operations
* **Reference:** 
  * https://www.geeksforgeeks.org/doubly-linked-list/

### 2. Circular Linked List
* **Structural Difference:** The `next` pointer of the last node (tail) points back to the `head` (first node), forming a closed loop instead of pointing to `NULL`. It can be implemented as either a singly or doubly linked list.
* **Use Case:** 
  * Operating system task scheduling
  * Multiplayer board games (passing the turn in a circle), 
  * Continuous image sliders on websites
* **Reference:**
  * https://www.geeksforgeeks.org/circular-linked-list/

### 3. Multi-linked List
* **Structural Difference:** A 2D (or n-D) linked list where nodes not only have a `next` pointer to continue the main list but also a `child` (or `down`) pointer that points to an entirely separate sub-linked list.
* **Use Case:** Hierarchical data 
  * Lists of lists
  * Sparse matrices
  * File systems (folders containing files/subfolders)
  * Multi-level dropdown menus
* **Reference:** 
  * https://www.geeksforgeeks.org/dsa/introduction-to-multi-linked-list/
  * https://webdocs.cs.ualberta.ca/~holte/T26/mlinked-lists.html

### 4. Stack (Linked List Implementation)
* **Structural Difference:** It is essentially a singly linked list with a **logical restriction**: insertions and deletions can only happen at **one end** (the head, usually called the `top`). It follows the **LIFO** (Last In, First Out) principle.
* **Use Case:** 
  * Undo mechanisms in text editors
  * Call stack management in programming languages
  * Evaluating mathematical expressions.
* **Reference:**
  * https://www.geeksforgeeks.org/implement-a-stack-using-singly-linked-list/

### 5. Queue (Linked List Implementation)
* **Structural Difference:** A linked list with a logical restriction where insertions happen at one end (the `tail`/`rear`) and deletions happen at the other end (the `head`/`front`). It follows the **FIFO** (First In, First Out) principle.
* **Use Case:** 
  * Patient waiting list
  * Printer task scheduling
  * Handling asynchronous data transfers.
* **Reference:**
  * https://www.geeksforgeeks.org/queue-linked-list-implementation/
