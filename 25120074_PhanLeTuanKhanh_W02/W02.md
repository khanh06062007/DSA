### Exercise 1A: Evaluate an expression tree using stack
Recursion is elegant, but every recursive call consumes memory on the system's Call Stack. If the expression tree is extremely deep (e.g., 10,000 chained additions), a recursive function will cause a Stack Overflow. Your challenge is to rewrite the evaluation function without using recursion.
### 1. Your task:
Implement the following function using `std::stack`:
``` cpp
int evaluateTreeIterative(TreeNode* root);
```
**Hints:**
- Search about "Iterative Post-order traversal" (Left $\rightarrow$ Right $\rightarrow$ Root). In this type of traversal, you must visit the root node twice before finally processing it.
- You may need 3 things: a `std::stack<TreeNode*>` to navigate the tree, a `std::stack<int>` to store your intermediate math results, and a `TreeNode* lastVisited` pointer to remember where you just came from.
- The workflow:
  1. Start at the root. Keep moving to the left child, pushing every node you pass into the `TreeNode*` stack, until you hit `nullptr`.
  2. When you can't go left anymore, look at (but do not pop) the top node of the stack. Let's call it `peekNode`.
  3. Check if `peekNode` has a right child, and if yes, check if you have visited it yet. If a right child exists but you haven't visited it, move your current pointer to that right child and go back to Step (i).
  4. Process node: If there is no right child, or you have already visited it, it's time to process `peekNode`:
      - If it's a number (a leaf): Convert `peekNode->val` to an integer and push it to your `stack<int>`.
      - If it's an operator (an internal node): Pop two numbers from your `stack<int>`, apply the operator (+, -, \*, /), and push the calculated result back into the stack<int>.
  5. Set `lastVisited = peekNode`, pop it from the `TreeNode*` stack, and repeat the process until the stack is empty.

### Exercise 3A: Equal product partition (Basic Backtracking)
#### 1. Problem Formulation
Given an array of $N$ strictly positive integers, determine if it is possible to partition the array into two disjoint subsets (Subset A and Subset B) such that:
1. Every element from the original array belongs to exactly one of the subsets.
2. The product of all elements in Subset A is exactly equal to the product of all elements in Subset B.

**Example 1:** `nums = [2, 4, 8]`
- Subset A: `[8]` $\rightarrow$ Product = 8
- Subset B: `[2, 4]` $\rightarrow$ Product = 2 * 4 = 8
- **Result:** `true`

**Example 2:** `nums = [1, 5, 3]`
- No matter how you split it, the products will never be equal.
- **Result:** `false`

#### 2. Your task
Write a backtracking function to explore all possible ways to divide the numbers.
``` cpp
bool backtrack(int* nums, int N, int index, long long prodA, long long prodB);
```

**Hints:**
- The entry point: There is a function called `canPartitionProduct`, which is already handled in your `main.cpp`. It already called this `backtrack` function starting with `index = 0` and both products initialized to `1`.
- Data types: Multiplying many integers grows exponentially fast. Use `long long` for `prodA` and `prodB` to prevent Integer Overflow. Start both products at 1.
- Base case: When `index == nums.size()`, you have assigned every number. Return true if `prodA == prodB`, otherwise false.
- The choices (Short-circuiting): For every number `nums[index]`, you have exactly two choices:
  - Multiply it to `prodA` and recurse.
  - Multiply it to `prodB` and recurse.
- Return true immediately if either of these recursive branches returns true (Use the `||` operator).

### Exercise 3B: Restore IP addresses (Advanced Backtracking)

#### 1. Problem Formulation
A valid IPv4 address consists of exactly four integers separated by single dots. Each integer is between 0 and 255 (inclusive) and cannot have leading zeros.

- Valid: `255.255.11.135`, `192.168.1.1`, `0.0.0.0`
- Invalid: `256.1.2.3` (256 too large), `192.168.01.1` (leading zero in 01)

Given a string `s` containing only digits, your task is to return all possible valid IP addresses that can be formed by inserting dots into `s`.

#### 2. Your task
Implement the backtracking function to find all valid IP combinations. 

``` cpp
void restoreIpAddresses(string s, int index, int partCount, string currentIP, string* result, int& resultCount);
```

**Hints:**
- The state: To keep track of your progress as you travel down the recursive tree, your helper function needs two special variables:
  1. `int partCount`: Tracks how many valid IP parts you have successfully formed (from 0 up to 4).
  2. `string currentIP`: The string that accumulates the parts you've chosen so far (e.g., `"192."`, `"192.168."`).
- Base case 1: If `partCount == 4` and `index == s.length()`, you have successfully used all characters to form exactly 4 parts. Add `currentIP` to `result[resultCount]` and increment `resultCount`.
- Base case 2: If `partCount == 4` but `index < s.length()` (too many characters left), or `index == s.length()` but `partCount < 4` (not enough characters), this path is invalid. return;
- The choices: From the current `index`, you can take 1, 2, or 3 characters to form a part.
  - Extract the substring using `s.substr(index, length)`.
  - Validate the substring: It must not be empty. If length > 1, it cannot start with '0'. Its integer value must be $\le 255$.
  - If valid, recursively call the function for `index + length`, `partCount + 1`, and append the part to `currentIP` (don't forget to add a dot `.` if it's not the last part).
