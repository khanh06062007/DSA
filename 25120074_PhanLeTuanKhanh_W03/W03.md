### Exercise 3A: The Hackathon Leaderboard

#### 1. Problem Formulation
A programming contest system stores team submissions in chronological order (earlier submission appears earlier in the array).

Each submission has this format:
```cpp
#include <string>
using namespace std;

struct Submission {
    string teamCode;
    int score;
};
```

You need to support two operations:
1. Sort the array in descending order by score.
2. Answer many score queries quickly.

Important tie rule:
- If two teams have the same score, the team submitted earlier must stay higher in rank.
- You are NOT allowed to add an extra field (such as timestamp/index) into `Submission`.

#### 2. Your Task
Implement these two functions:
```cpp
void sortLeaderboard(Submission subs[], int left, int right);
string queryScore(Submission subs[], int n, int target);
```

Parameter explanation:
- `subs`: array of submissions to be processed.
- `left`: left boundary index (inclusive) of the subarray to sort.
- `right`: right boundary index (inclusive) of the subarray to sort.
- `n`: total number of submissions in `subs`.
- `target`: score value to look up.

Requirements:
- `sortLeaderboard` must produce a **STABLE** descending order by score.
- `queryScore` returns the `teamCode` of the highest-ranked team with exactly `target` score.
- If no team has that score, return `"Not found"`.
- Each query must run in `O(log n)`.

#### 3. Input & Output Format
**Input file:**
- Line 1: integer `N` (`1 <= N <= 100000`) - number of submissions.
- Next `N` lines: `teamCode score`.
- Next line: integer `Q` (`1 <= Q <= 100000`) - number of queries.
- Next `Q` lines: one integer `target` score per line.

**Output file:**
- Print `Q` lines.
- Each line is either a `teamCode` or `Not found`.

#### 4. Hints
- For stable descending sort, use a stable sorting algorithm.
- Consider the time complexity required for N = 100,000, and the stable property when choosing your sorting algorithm.
- Remember to adjust the if-else conditions for binary search on a descending array.

#### 5. Submission & Setup Requirements
- Directory structure: inside `W03_E03A`, submit exactly one source file `main.cpp`.
- Allowed libraries: `<iostream>`, `<fstream>`, `<string>`.
- Program must accept exactly two CLI arguments: input path and output path.

See `W03_E03A/main.cpp`.

### Exercise 3B: Capacity To Ship Packages Within D Days

#### 1. Problem Formulation
You have `n` packages arranged on a conveyor belt. `weights[i]` is the weight of the `i`-th package.

Each day, you load packages onto a ship in the same order as given. You cannot reorder packages. On each day, the total loaded weight must not exceed the ship capacity.

Given integer `D`, return the minimum ship capacity so that all packages are shipped within `D` days.

#### 2. Your Task
Implement:
```cpp
int minimumCapacityPerDay(int weights[], int n, int days);
```

Parameter explanation:
- `weights`: array of package weights in the original shipping order.
- `n`: number of packages in `weights`.
- `days`: maximum number of days allowed to ship all packages.

Expected approach:
- Binary search on the answer (capacity).
- Check feasibility of a candidate capacity in linear time.

#### 3. Input & Output Format
**Input file:**
- Line 1: two integers `n` and `D` (`1 <= D <= n <= 50000`).
- Line 2: `n` integers `weights[i]` (`1 <= weights[i] <= 500`).

**Output file:**
- A single integer: the minimum required capacity.

#### 4. Hints
- Lower bound of capacity = max single package weight.
- Upper bound of capacity = sum of all weights.
- If the package loading can finish in `<= D` days, try to decrease capacity.
- Otherwise, increase capacity.

#### 5. Submission & Setup Requirements
- Directory structure: inside `W03_E03B`, submit exactly one source file `main.cpp`.
- Allowed libraries: `<iostream>`, `<fstream>`.
- Program must accept exactly two CLI arguments: input path and output path.

### Exercise 3C: Sort Colors (Dutch National Flag)

#### 1. Problem Formulation
Given an array `nums` of size `n`, each element is one of `0`, `1`, `2`:
- `0` = red
- `1` = white
- `2` = blue

Reorder the array in-place so that all red objects come first, then white, then blue.

This exercise is the classic **Dutch National Flag** partitioning problem.

#### 2. Your Task
Implement:
```cpp
void sortColors(int nums[], int n);
```

Parameter explanation:
- `nums`: input array to be sorted in-place.
- `n`: number of elements in `nums`.

#### 3. Input & Output Format
**Contraints:**
- You must satisfy all of the following:
  - In-place algorithm (`O(1)` extra space).
  - Exactly **ONE** full pass over the array (`O(n)` time).
  - You must rearrange the elements using ONLY swap operations. Direct value assignment (like `nums[i] = 0;`) is strictly forbidden.
  - Do NOT use counting variables/arrays to count frequencies.

**Input file:**
- Line 1: integer `n` (`1 <= n <= 1000000`).
- Line 2: `n` integers `nums[i]`. `nums[i]` is always one of `0`, `1`, `2`.

**Output file:**
- One line containing the sorted array values, separated by one space.

#### 4. Hints
- Maintain three regions using pointers (`low`, `mid`, `high`).

#### 5. Submission & Setup Requirements
- Directory structure: inside `W03_E03C`, submit exactly one source file `main.cpp`.
- Allowed libraries: `<iostream>`, `<fstream>`.
- Program must accept exactly two CLI arguments: input path and output path.
